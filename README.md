# 智能控制面板 Web 应用

这是一个基于 Web 技术实现的智能控制面板，用于模拟工业自动化系统中的设备控制和检测功能。本应用是从 PyQt5 桌面应用转换而来的 Web 版本，最大程度地还原了原始界面的视觉效果和功能。

## 功能特点

- **视频监控与检测**：支持通过摄像头或上传图片进行颜色和角度检测
- **实时数据显示**：展示检测结果，包括颜色识别、角度测量和误差值
- **串口通信模拟**：模拟与硬件设备的串口通信功能
- **电机控制**：支持电机启动和复位操作
- **智能操作流程**：模拟自动化生产流程中的智能操作
- **冲压操作控制**：模拟冲压设备的操作控制
- **系统日志记录**：记录所有操作和通信日志
- **平台信息展示**：显示系统运行环境信息

## 技术栈

- **HTML5**：页面结构和语义化标签
- **CSS (Tailwind CSS)**：响应式布局和样式
- **JavaScript**：交互逻辑和模拟功能实现
- **Font Awesome**：图标库
- **Canvas API**：绘制检测结果和视频覆盖层
- **WebRTC (getUserMedia)**：访问用户摄像头
- **LocalStorage**：本地存储检测记录

## 快速开始

### 本地运行

1. 克隆或下载此仓库到本地计算机
2. 找到并打开 `smart_manufacturing_web.html` 文件
3. 在浏览器中打开此文件（推荐使用 Chrome、Firefox 或 Edge 等现代浏览器）

```bash
# 在浏览器中打开示例
file:///path/to/smart_manufacturing_web.html
```

### GitHub Pages 部署

1. **准备仓库**
   - 创建一个新的 GitHub 仓库，命名为 `smart-manufacturing-web` 或其他您喜欢的名称
   - 将 `smart_manufacturing_web.html` 文件和 `README.md` 文件上传到仓库中

2. **启用 GitHub Pages**
   - 在 GitHub 仓库页面中，点击顶部的 **Settings** 选项卡
   - 向下滚动找到 **Pages** 部分
   - 在 **Branch** 下拉菜单中，选择 **main** 或 **master** 分支
   - 在 **folder** 下拉菜单中，选择 **/(root)** 选项
   - 点击 **Save** 按钮保存设置

3. **访问部署的页面**
   - GitHub Pages 通常需要几分钟时间来部署您的页面
   - 部署完成后，您将在 **Pages** 部分看到一个绿色的成功消息，包含访问链接
   - 链接格式通常为：`https://yourusername.github.io/your-repository-name/`

## 功能使用说明

### 视频监控与检测

1. **打开摄像头**：点击「打开摄像头」按钮，授权浏览器访问您的摄像头
2. **关闭摄像头**：点击「关闭摄像头」按钮，停止摄像头访问
3. **插入图片**：点击「插入图片」按钮，上传本地图片进行检测
4. **检测图片**：点击「检测图片」按钮，对当前显示的图片进行颜色和角度检测

### 串口通信

1. **选择串口**：从下拉列表中选择要连接的模拟串口
2. **打开串口**：点击「打开串口」按钮，建立串口连接
3. **关闭串口**：点击「关闭串口」按钮，断开串口连接

### 控制操作

1. **启动电机**：点击「启动电机」按钮，控制电机开始运行
2. **电机复位**：点击「电机复位」按钮，将电机重置到初始位置
3. **智能操作**：点击「智能操作」按钮，执行自动化流程
4. **冲压操作**：点击「冲压操作」按钮，执行冲压工序

### 检测记录

- 每次成功检测后，结果会自动保存在本地存储中
- 可以在「识别结果」区域查看最近的检测结果

## 注意事项

1. **摄像头权限**：使用摄像头功能时，需要在浏览器中授予摄像头访问权限
2. **串口通信**：当前实现为模拟串口通信，不连接实际硬件设备
3. **性能要求**：在处理高清视频流时，可能需要较高的设备性能
4. **浏览器兼容性**：建议使用 Chrome、Firefox 或 Edge 等现代浏览器以获得最佳体验
